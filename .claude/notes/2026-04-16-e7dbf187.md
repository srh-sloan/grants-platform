## Learnings

- **P4.1 proves the architecture:** Adding a second grant required ZERO Python
  changes. The seed.py loader, form runner, applicant routes, assessor routes,
  and scoring engine all handled Common Ground without modification.
- **Seed weight validator limitation:** `seed.py` uses integer comparison
  `weights != 100`. This means 12.5% weights must be rounded to integers.
  Consider switching to `abs(weights - 100) < 0.01` for float tolerance.
- **Gate data in JSON columns:** Using underscore-prefixed keys in scores_json
  is a pragmatic pattern that avoids model changes. The scoring engine ignores
  them because it only reads criterion IDs from config.
- **Server-side conditional fields:** The MVP approach (re-render on submit)
  is simple and works. Progressive enhancement with JS could improve UX later.
