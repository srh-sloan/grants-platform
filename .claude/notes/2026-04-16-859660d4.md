# Session Note — 2026-04-16

## Task
Full UX journey audit of the grants platform (`/ux-journey` skill).

## What was done
- Explored every route, template, form, nav partial, and auth flow in the codebase.
- Produced a structured UX audit covering both the applicant and assessor journeys.
- No code was changed — this was a pure analysis/planning session.

## Key findings

### P0 Blockers (must fix before live demo)
1. **No nav for unauthenticated users** — `partials/header.html` sets `_nav_items = []` for anonymous visitors. No Sign in / Create account links anywhere on the site. Users are dead-ended on the homepage.
2. **Landing page has no CTAs** — `public/index.html` lists grants with slug debug text and no "Apply" buttons. Empty state says "Run python seed.py".
3. **Date fields don't pre-fill** — `forms/page.html` renders `govukDateInput` without passing saved day/month/year values back into `items`. Data appears lost on return visits.

### P1 Friction
4. No application reference number shown after submission.
5. Assessor scoring errors are flashed one-per-criterion rather than grouped into one error summary.
6. Assessor detail page is very dense — AI assessment + scoring form + moderation + decision all in one scrollable right column.
7. Decision form re-notifies applicant without confirmation when decision already recorded.
8. AI score column shows raw number without denominator (e.g., "187" instead of "187 / 300").
9. No manual scoring status indicator in the assessor queue.
10. Register page gives no guidance to assessors who can't self-register.
11. Dashboard has a duplicate "Sign out" button (also in header nav).

### P2 Polish
12. Progress indicator on form pages is small and grey (`govuk-body-s`).
13. Application detail shows field IDs converted to title case rather than actual field labels.
14. Error pages (403/404/500) have minimal recovery guidance.
15. `govuk-!-background-colour-red` on allocation page likely doesn't exist in vendored CSS.
16. Admin "Create user" gives no hint that password must be shared out-of-band.
17. Review page "Back to your applications" is inside `govuk-button-group` rather than a proper `govuk-back-link`.

## Decisions / notes for future sessions
- The core GOV.UK macro usage, error summary patterns, and multi-step form flow are all correct — only the entry-point and a few detail issues need work.
- The date pre-fill bug (item 3) is the most likely source of applicant confusion and should be verified with `forms_runner.py` to confirm how date sub-fields are assembled on POST.
- The assessor detail page restructure (tabs or anchor navigation) would benefit from a brief design discussion before implementation — the current single-page approach may be intentional for screen-reader linearity.
