About the CSV format

Each row has columns: type, key, value, extra1, extra2, extra3

  • meta rows set grant-level fields (name, slug, budget, dates, contact email).
  • eligibility rows define pass/fail rules. value is the rule type (in, equals, max, min); extra1 is the threshold or pipe-separated allowed values; extra2 is the human-readable rule label.
  • criterion rows define scoring criteria. value is the criterion label; extra1 is the weight (integer, all weights must sum to 100); extra2 is the max score (usually 3); extra3 is true if a zero score triggers auto-rejection.
  • Rows starting with # in the type column are treated as comments.
